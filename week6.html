Technology Research Paper: Dropbox

       Dropbox, founded in 2007 by Drew Houston and Arash Ferdowsi, has grown to over 300 million users across the world in only eight years. It is the freemium service that allows people to quickly and easily upload and share photos, documents, and videos (Dropbox, 2015). Even with this exponential growth, Dropbox has managed to still operate with a team of less than 100, spread out between three different cities. There are tens of thousands of servers responsible for storing and serving the metadata for every file in Dropbox. Additionally, one billion files are saved every day to Dropbox; so in order for the service to be reliable, their servers need to be fast and consistent (Cook, 2014). Dropbox has their technology stack well figured out, and startup companies everywhere should take note.

Amazon Web Services

       Dropbox is currently using Amazon’s collection of compatible “pay for what you use”, cloud infrastructure web services, known as AWS. AWS is convenient because it allows the user to only pay for the data or storage that they actually use. AWS offers an assortment of services, each of which can be paid for individually. Dropbox, for instance, operates one of the largest global deployments into S3, runs tens of thousands of EC2 instances, and heavily utilizes both the SQS and Route 53 services (Cook, 2014). Dropbox’s features, such as Carousel, Microsoft Office document previews, full-text search, and Mailbox, all rely on AWS “to perform operations for every Dropbox file update in near-realtime” (Cook, 2014).

       Amazon Simple Storage Service “provides developers and IT teams with secure, durable, highly-scalable object storage. Amazon S3 is easy to use, with a simple web services interface to store and retrieve any amount of data from anywhere on the web” (Amazon Web Services, Inc., 2015). Amazon S3 gives the user the option of using it alone, or pairing it with the other AWS services (Amazon Web Services, Inc., 2015).

       Amazon Elastic Compute Cloud “provides resizable compute capacity in the cloud. It is designed to make web-scale cloud computing easier for developers.” EC2 also “reduces the time required to obtain and boot new server instances to minutes, allowing you to quickly scale capacity, both up and down, as your computing requirements change” (Amazon Web Services, Inc., 2015).

       Amazon Simple Queue Service “is a fast, reliable, scalable, fully managed message queuing service. SQS makes it simple and cost-effective to decouple the components of a cloud application. You can use SQS to transmit any volume of data, at any level of throughput, without losing messages or requiring other services to be always available” (Amazon Web Services, Inc., 2015).

       Amazon Route 53 is “a highly available and scalable cloud Domain Name System web service” (Amazon Web Services, Inc., 2015). It connects user requests to the other infrastructures running in AWS but can also route to any infrastructure outside of AWS (Amazon Web Services, Inc., 2015).

Python

       99.9% of the code used for Dropbox is written with Python. Python allows Dropbox to work no matter what file system, OS, or application the user accesses it on. Python was also their best option for quickly working through the error cases they receive on the multiple platforms Dropbox supports (Hoff, 2011).

       The only real problem that Dropbox encounters with Python is memory usage. Python was using up too much memory, causing Dropbox to have memory optimization issues. The problem was memory fragmentation, which is “what happens when different sized memory blocks are continually being deleted and allocated. Contiguous blocks of memory can no longer be allocated” (Hoff, 2011). As a solution, Dropbox created a custom memory allocator written in C.

SSL

       Dropbox is engaged in open communication with their users in regards to their privacy and security on their blog. In order to keep the information you send to Dropbox safe and secure, SSL is used for all encryption methods. SSL “keeps people from snooping on your stuff, regardless of what device you’re using to access your files or if you’re on public WiFi” (Akhawe, 2015). In 2014, Dropbox became aware of multiple bugs in SSL. This prompted them to not only make a few patches and change some settings, but to also disable the SSLv3 protocol. These updates allowed Dropbx to become “one of the very few major sites to get an A+ rating from SSL Labs” (Akhawe, 2015).
